# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述

### 背景與動機
現代人生活節奏緊湊，往往容易忽略日常的開銷與收入情況，導致月底經常面臨「錢都花去哪了」的困擾。為了解決這個問題，我們需要一個簡單、直覺的個人記帳簿系統，幫助使用者輕鬆紀錄每一筆收入與支出，並透過直觀的介面掌握個人的財務狀況。

### 目標用戶
需要記帳的人，包括但不限於：學生、上班族、家庭主婦，以及任何希望有效管理個人財務、追蹤金錢流向的使用者。

### 核心價值主張
提供一個快速且易用的記帳平台，讓使用者能輕鬆紀錄日常收支、即時查看餘額，並透過金錢流向統計掌握自身消費習慣，進而達成財務管理的目標。

## 2. 功能需求

### 主要功能與使用者故事

1. **紀錄收支日期**
   - 作為 **一般使用者**，我希望 **在新增一筆紀錄時能指定日期**，以便 **我能準確追蹤每一天發生了什麼開銷或收入**。

2. **輸入收支金額**
   - 作為 **一般使用者**，我希望 **能輸入具體的收入或支出金額**，以便 **系統能正確計算我的總資產與帳務變化**。

3. **分類支出與收入項目**
   - 作為 **一般使用者**，我希望 **能為每一筆帳務選擇對應的項目或類別（如餐飲、交通、薪水等）**，以便 **日後能清楚知道我的錢花在哪些地方**。

4. **顯示目前餘額**
   - 作為 **一般使用者**，我希望 **在系統首頁就能看到目前的總餘額統計**，以便 **隨時掌握我還有多少可動用的資金**。

5. **統計金錢流向**
   - 作為 **一般使用者**，我希望 **能查看一段時間內的總收支統計與金錢流向（如報表或簡單的統計數字）**，以便 **分析我的消費習慣並進行財務規劃**。

## 3. 非功能需求

### 技術限制
- 後端框架：Flask (Python)
- 前端樣式：Jinja2 模板引擎與 HTML / CSS
- 資料庫：SQLite (適合個人單機或輕量級網站使用)

### 效能與安全考量
- **效能**：系統操作需流暢，新增與查詢紀錄的反應時間應小於 1 秒。
- **安全**：未來若考慮多人使用，需加入基本的使用者登入功能；輸入欄位需防範 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

| 功能項目 | 層級 | 說明 |
| -------- | ---- | ---- |
| 新增收支紀錄（含日期、金額、項目） | Must Have | 系統最核心的記帳功能 |
| 顯示目前總餘額 | Must Have | 提供使用者即時的財務總覽 |
| 收支明細列表 | Must Have | 讓使用者能查看所有已紀錄的帳目 |
| 金錢流向簡單統計（總收入、總支出） | Should Have | 提供基本的月度或自訂期間統計數字 |
| 收支項目分類管理（新增/修改類別） | Should Have | 讓使用者自訂符合自己習慣的分類 |
| 圖表化金錢流向（圓餅圖/長條圖） | Nice to Have | 更直覺的財務數據視覺化呈現 |
| 匯出記帳資料 (如 CSV) | Nice to Have | 讓使用者能備份資料至其他軟體使用 |
